사용한 데이터 (6가지)
  1. 산업별 임금 및 근로시간 데이터
  2. 산업별 취업자 수 데이터
  3. 성별, 연령별 취업자 수 데이터
  4. 성별, 연령별 실업자 수 데이터
  5. 비경제활동인구 중 '쉬었음' 수 데이터
  6. 비경제활동인구 '쉬었음' 이유 데이터


주요 정제 단계 (Step-by-Step):
Step 1: 연령대 필터링 및 범위 확정(KOSIS에서 행렬 선택 및 직접 변환)
수행 내용: 원천 데이터에서 전 연령대 데이터를 제외하고, 분석 대상인 15-29세(청년층) 데이터만을 추출함
이유: AI 로 인해 '쉬었음' 청년이 대두되고 있는 지금, 청년층이 겪는 고유한 고용 시장의 특성(첫 직장 진입, 구직 단념 사유 등)을 심층 분석하기 위함

Step 2: 시점(Date) 데이터 표준화(엑셀로 직접 변환)
수행 내용: 2025.07 형태의 텍스트 데이터를 2025-07-01 형태의 표준 날짜 형식(ISO 8601)으로 변환함
효과: BI 도구(Streamlit)에서 날짜 데이터로 인식하게 하여 월별 추이 및 증감률 계산의 편의성 확보함

Step 3: 산업 분류 체계 및 성별 코드 통일(엑셀로 직접 변환)
수행 내용: 각 테이블에 흩어져 있는 산업 명칭을 표준 산업 분류 코드(A, B, C...)로 매핑하여 통일함
이유: 서로 다른 소스에서 온 '산업별 취업자'와 '산업별 임금' 테이블을 결합하기 위한 공통 키(Join Key) 설정함

Step 4: 파생 변수 생성(엑셀로 직접 수행)
수행 내용: 
  1. 시간당 임금: 전체임금총액 / 전체근로시간을 통해 실질적인 근로 여건을 파악할 수 있는 지표 생성함
  2. 실업률: (실업자 / 경제활동인구) * 100 수식을 적용하여 고용 안전성을 수치화함

Step 5: 결측치(Null) 처리 전략 수립(SQL 문 사용하여 확인)
수행 내용: 농업(A), 공공행정(O) 등 임금 정보가 비어 있는 산업군 확인함
대응: 인위적인 추정치 대입 대신, 취업자 현황 분석에는 전체 산업을 포함하되 임금 분석 시에는 데이터가 존재하는 주요 산업군으로 한정하여 데이터의 신뢰도 유지함


최종 DB 스키마 설계:
  1. 경제활동현황 : 성별/시점별 취업자, 실업자, 실업률	고용 지표 KPI용
  2. 산업별_취업자 : 산업별 청년 취업자 수	-> 메인 분석 축
  3. 비경제활동인구 : 구직활동을 하지 않는 청년 규모	-> 잠재적 고용 위기 파악
  4. 쉬었음의_주된_이유 : 구직 단념 및 휴식의 세부 사유 -> 심층 원인 분석용
  5. 산업별_임금_및_근로시간 : 산업별 평균 근로 환경 데이터 -> 고용의 질 분석 (선별적 활용)


분석 시나리오 및 기대 효과:
시나리오: 하반기(7월~12월) 동안 청년 실업률의 변동과 '쉬었음' 사유 중 '원하는 일자리 없음'의 상관관계 분석함
기대 효과: 단순한 수치 나열이 아닌, 청년층이 특정 산업군에 쏠리는 현상과 그들의 자발적/비자발적 노동 시장 이탈 원인을 데이터로 증명함
